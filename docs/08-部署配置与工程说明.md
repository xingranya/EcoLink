## 4. application.yml 配置说明

### 4.1 数据源配置

```yaml
spring:
  datasource:
    url: ${DB_URL:jdbc:mysql://localhost:3306/ecolink?...}
    username: ${DB_USERNAME:root}
    password: ${DB_PASSWORD:root}
```

说明：

- 优先读取环境变量
- 未提供时使用本地默认值

### 4.2 JPA 配置

```yaml
spring:
  jpa:
    open-in-view: false
    hibernate:
      ddl-auto: validate
```

工程意义：

- `open-in-view: false` 控制延迟加载边界
- `ddl-auto: validate` 表示表结构交由 Flyway 管理

### 4.3 Flyway 配置

```yaml
spring:
  flyway:
    enabled: true
    locations: classpath:db/migration
```

说明系统启动后会自动校验并执行数据库迁移脚本。

### 4.4 JWT 配置

```yaml
app:
  jwt:
    issuer: ecolink
    secret: ${JWT_SECRET:...}
    expire-hours: 24
```

说明：

- JWT 发行者固定为 `ecolink`
- 默认有效期为 24 小时
- 密钥可通过环境变量覆盖

### 4.5 CORS 配置

```yaml
app:
  cors:
    allowed-origins: ${CORS_ALLOWED_ORIGINS:http://localhost:3000,http://localhost:5173}
```

说明：

- 默认支持本地前端常用端口跨域访问
- 适合前后端分离开发

## 5. 目录结构与部署边界

```text
EcoLink
├─src/                 # Vue 前端源码
├─server/              # Spring Boot 后端源码
├─docs/                # 项目文档与技术 wiki
├─dist/                # 前端构建结果
├─package.json         # 前端配置
└─server/pom.xml       # 后端配置
```

从结构上看，前后端可独立构建、独立部署。

## 6. 部署思路

当前项目适合采用如下部署方式：

1. 前端通过 Vite 构建为静态资源
2. 后端以独立 Spring Boot 服务运行
3. MySQL 独立部署
4. 使用 Nginx 反向代理前端静态资源与后端 API

## 7. 论文可直接引用的部署描述

> 系统部署采用典型前后端分离方式：前端构建为静态资源，由 Web 服务器提供访问；后端以 Spring Boot 应用形式独立运行；数据库采用 MySQL 持久化存储；系统配置通过环境变量注入，并借助 Flyway 完成数据库结构初始化与版本演进，从而实现了部署过程的标准化和环境切换的便利性。
