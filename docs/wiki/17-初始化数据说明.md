# 17. 初始化数据说明

> 文档类型：项目环境文档  
> 同步时间：2026-03-13  
> 目标：明确初始化数据、默认账号以及由 Flyway 自动生成的内容

## 1. 初始化方式

项目数据库初始化由 Flyway 执行，主要包括：

1. `V1__schema.sql`：创建用户、商品、购物车、订单等核心表。
2. `V2__seed.sql`：写入初始分类、商品、商品图片、普通用户、默认地址和购物车数据。
3. `V3__admin_role.sql`：新增角色字段并写入管理员账号。
4. `V4__fix_admin_password.sql`：修正管理员密码格式，保证与前端登录流程兼容。

## 2. 默认账号

- `demo` 用户初始化时已经拥有一条默认收货地址。
- `demo` 用户初始化时已写入两条购物车数据，可用于快速体验订单创建流程。

## 4. 初始化检查建议

### 4.1 最稳妥方式

1. 清空或重建本地数据库。
2. 重新执行 Flyway 迁移。
3. 确认默认账号可以登录。
4. 用普通用户先跑一遍“搜索 -> 加购 -> 下单 -> 支付”链路。
5. 用管理员再跑一遍“后台仪表盘 -> 订单详情 -> 发货/完成”链路。

### 4.2 如果数据已经被操作改动

优先采用“重新迁移数据库”的方式恢复，而不是手工改表。  
原因是 Flyway 迁移具备更好的可重复性，也更容易说明环境初始化流程。
