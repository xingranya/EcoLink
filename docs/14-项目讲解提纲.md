# 14. 项目讲解提纲

> 文档类型：项目说明文档  
> 推荐时长：5-8 分钟  
> 使用方式：可按时间线介绍项目，也可按模块拆分说明

## 1. 0:00 - 0:40 项目背景

讲解要点：

- 本项目名称为 EcoLink，是一个面向生态农产品场景的前后端分离电商系统。
- 设计目标不是做完整商业平台，而是围绕生态农产品场景实现完整的用户交易链路与后台运营链路。
- 用户端关注商品浏览、搜索、下单和支付，管理员端关注商品运营、库存观测和订单履约。

## 2. 0:40 - 1:30 技术架构

讲解要点：

- 前端使用 Vue 3 + TypeScript + Pinia + Vue Router。
- 后端使用 Spring Boot + Spring Security + JPA + Flyway。
- 认证方式采用 JWT，实现前后端分离下的身份识别。
- 数据库使用 MySQL，结构与初始数据通过 Flyway 自动初始化。

推荐查看页面：

- 首页
- Wiki 中的架构图

## 3. 1:30 - 3:40 前台用户链路

推荐顺序：

1. 首页  
说明首页不仅提供商品入口，还展示生态溯源、品质保障和完整业务闭环。

2. 搜索页  
说明关键字搜索、分类筛选和排序能力。

3. 商品详情页  
说明商品详情除了价格和库存，还展示产地、销量、推荐理由和溯源批次。

4. 购物车页  
说明数量修改、金额汇总和库存校验。

5. 创建订单与支付页  
使用默认账号 `demo / 123456` 完成下单和模拟支付，强调支付成功后的状态反馈和自动跳转。

6. 我的订单页  
说明订单进度条，以及支付时间、发货时间、完成时间等状态字段。

## 4. 3:40 - 5:30 后台管理员链路

推荐顺序：

1. 使用 `admin / admin123` 登录。
2. 进入后台仪表盘，介绍商品总数、低库存预警、累计营收、最近订单和热销商品。
3. 打开商品管理页，说明后台支持新增、编辑、上下架与条件筛选。
4. 打开订单管理页，展开订单详情抽屉，展示订单明细和收货信息。
5. 点击“发货”或“完成”，再回到前台订单页说明状态同步变化。

## 5. 5:30 - 6:30 技术亮点总结

可直接使用以下表述：

1. 系统采用前后端分离架构，前端和后端通过 REST API 解耦。
2. 使用 JWT 和角色控制实现用户端与管理端权限隔离。
3. 通过事务处理购物车、订单、库存和销量回写，保证核心交易链路的一致性。
4. 使用 Flyway 管理表结构与种子数据，使项目环境更容易复现。
5. 通过 Wiki、字段字典、测试文档和项目讲解提纲补充系统说明材料。

## 6. 6:30 - 7:00 结束陈述

推荐结尾：

“EcoLink 完成了从商品展示、搜索下单、模拟支付到后台订单履约的完整闭环，同时具备清晰的技术架构、数据库设计、接口安全和测试支撑，能够完整体现生态农产品电商系统的核心业务与工程实现。”
