# 接口字段与返回模型字典

> 文档定位：以字段级形式说明关键接口的请求体、响应体与分页结构  
> 同步依据：DTO、Controller、通用响应模型、后台仪表盘接口  
> 推荐用途：毕业论文“接口详细设计”章节

## 1. 通用响应模型

统一响应定义如下：

```java
public record ApiResponse<T>(
        int code,
        String message,
        T data,
        LocalDateTime timestamp
) {}
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `code` | `int` | 业务状态码 |
| `message` | `String` | 返回说明 |
| `data` | `T` | 业务数据 |
| `timestamp` | `LocalDateTime` | 响应时间 |

## 2. 通用分页模型

前台商品接口使用：

```java
public record PageResult<T>(
        List<T> list,
        int page,
        int size,
        long total
) {}
```

字段说明：

| 字段 | 说明 |
|---|---|
| `list` | 当前页数据列表 |
| `page` | 当前页码 |
| `size` | 页大小 |
| `total` | 总记录数 |

后台管理分页模型直接返回：

- `content`
- `totalElements`
- `totalPages`
- `number`

## 6. 订单接口字段

### 6.1 创建订单请求

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `addressId` | `Long` | 非空 | 地址 ID |
| `cartItemIds` | `List<Long>` | 可选 | 购物车条目 ID 集合 |

### 6.2 订单响应

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | `Long` | 订单 ID |
| `orderNo` | `String` | 订单编号 |
| `status` | `OrderStatus` | 订单状态 |
| `totalAmount` | `BigDecimal` | 订单总金额 |
| `receiverName` | `String` | 收货人 |
| `receiverPhone` | `String` | 收货电话 |
| `receiverAddress` | `String` | 收货地址 |
| `paidAt` | `LocalDateTime` | 支付时间 |
| `shippedAt` | `LocalDateTime` | 发货时间 |
| `completedAt` | `LocalDateTime` | 完成时间 |
| `createdAt` | `LocalDateTime` | 下单时间 |
| `items` | `List<OrderItemResponse>` | 订单明细 |

订单明细项字段包括：

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | `Long` | 明细 ID |
| `productId` | `Long` | 商品 ID |
| `productName` | `String` | 下单时商品名称快照 |
| `productImage` | `String` | 下单时商品图片快照 |
| `salePrice` | `BigDecimal` | 下单时成交单价 |
| `quantity` | `Integer` | 购买数量 |
| `subtotal` | `BigDecimal` | 小计金额 |

### 6.3 订单状态枚举

| 枚举值 | 含义 |
|---|---|
| `UNPAID` | 待支付 |
| `PAID` | 已支付 |
| `SHIPPED` | 已发货 |
| `COMPLETED` | 已完成 |
| `CANCELLED` | 已取消 |

## 9. 后台订单详情返回模型

后台 `GET /api/v1/admin/orders/{id}` 的 `data` 结构如下：

| 字段 | 类型 | 说明 |
|---|---|---|
| `order` | `Object` | 订单头信息 |
| `items` | `List<Object>` | 订单商品明细 |

其中 `order` 对象包含：

- `id`
- `orderNo`
- `status`
- `totalAmount`
- `receiverName`
- `receiverPhone`
- `receiverAddress`
- `createdAt`
- `paidAt`
- `shippedAt`
- `completedAt`

其中 `items` 条目包含：

- `id`
- `productName`
- `productImage`
- `salePrice`
- `quantity`
- `subtotal`

## 10. 论文可直接引用的接口描述

> 系统接口设计采用 DTO 建模方式，以字段级形式对请求输入与响应输出进行约束和描述。统一响应模型使得不同接口在结构上保持一致，而分页模型和后台统计模型则针对不同业务场景进行了专门设计，从而满足商城前台展示和后台管理分析的双重需求。
